# 修复说明：输出文件夹锁定问题

## 问题描述
用户报告 image-text-remover 工具在生成失败时显示"输出文件夹已打开"的错误。这个问题发生在处理完成后尝试自动打开输出目录时。

## 根本原因分析

1. **文件锁定冲突**：Windows Explorer 打开文件夹时可能会锁定某些文件操作
2. **时序问题**：后台线程完成文件写入后，主线程立即尝试打开文件夹
3. **命令差异**：使用 `explorer.exe` 命令相比 `start` 命令更容易出现兼容性问题

## 解决方案

### 1. 改进文件夹打开逻辑 (`main_window.py`)

**修改前：**
```python
def _open_output_dir(self):
    """打开输出目录"""
    import os
    import subprocess

    output_path = Path(self.output_dir.get())
    output_path.mkdir(parents=True, exist_ok=True)

    if os.name == 'nt':  # Windows
        subprocess.run(['explorer', str(output_path)])
    else:  # macOS/Linux
        subprocess.run(['open', str(output_path)])
```

**修改后：**
- 添加重试机制（最多3次重试）
- 使用 `start` 命令替代 `explorer` 命令（更稳定）
- 添加超时处理（5秒超时）
- 改进错误处理和日志记录
- 失败时显示输出目录路径作为备选方案

### 2. 改进文件写入逻辑 (`worker.py`)

**修改前：**
```python
def _generate_report(self):
    """生成JSON处理报告"""
    try:
```

**修改后：**
- 添加文件写入重试机制
- 专门处理 `PermissionError`（文件锁定）
- 改进错误日志和重试提示

### 3. 添加处理完成延迟 (`main_window.py`)

在 `_on_finished_ui()` 中添加 1 秒延迟，确保所有文件操作完成后再打开文件夹：

```python
# 等待一小段时间确保所有文件操作完成
time.sleep(1)

# 自动打开输出文件夹（如果有成功处理的图片）
if done_count > 0:
    try:
        self._open_output_dir()
    except Exception as e:
        self._log(f"[警告] 打开输出目录失败: {e}")
```

## 使用建议

1. **如果仍然遇到文件锁定问题**：
   - 手动关闭已打开的输出文件夹
   - 在工具中选择不同的输出目录
   - 查看日志中的输出路径，手动导航到该目录

2. **错误处理改进**：
   - 所有文件操作都有重试机制
   - 失败时会显示详细的错误信息和输出路径
   - 日志中会记录每次重试的尝试
